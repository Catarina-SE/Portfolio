Key Features

Interface-Based Architecture: Clean C++ design with IInteractable interfaces and component-based systems for maintainable, extensible code
Hybrid Grab System: 5-iteration development process resulting in intelligent switching between snap-to-grab-points and free-form object manipulation
Physics Constraint Integration: Weight-based constraints requiring both hands for heavy objects, with tuned strength parameters for realistic manipulation
Event-Based Communication: Broadcast event system prevents dependencies between game systems while maintaining clear component interactions
Motion Sickness Prevention: Multiple locomotion options with real-time comfort adjustments - prioritizing player comfort over complexity
